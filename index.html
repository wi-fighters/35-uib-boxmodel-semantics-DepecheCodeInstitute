This page is for my practice using valid semantic elements, with a particular nod to "a11y". For more on web accessibility, see also a11yproject).

Knowledge and tools with which I would like to experiment:

  • proper use of semantic elements, i.e. nav bar, button, aside, article, input labels NB: built-in HTML elements often have native keyboard accessibility -> move info to appropriate section
  • checkboxes, apparently
  • using descriptive anchor txt (rather than just "click me")
  • using alt text and titles for contextually important imgs
  • using css for decorative-only images
  • inform users when certain events trigger, i.e. when a new window will be opened, i.e. "(opens in new tab/powerpoint)", etc
  • use meaningful text labels for input fields
  • use ARIA attribute values when native HTML features are not available, particular with nested divs
  • practise using css to style element
